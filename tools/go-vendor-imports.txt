# Bx-12 Go -- the std packages the browser payload ships export data for.
#
# WHY A LIST AT ALL
#   The browser has no cmd/go, so std cannot be built on demand: every package a
#   user's practice.go might import must have its .a archive vendored ahead of
#   time (docs/go-self-hosted.md, sections 3 and 5). Ship too little and a
#   perfectly ordinary "import sort" fails to compile. Ship all of std and the
#   payload carries net/http, crypto, go/parser and database/sql for no reason.
#
# HOW TO CHANGE IT
#   Add the import path, re-run the vendor step, read the new number. The
#   closure is transitive -- adding "sort" also pulls what sort needs -- so the
#   marginal cost of a line is rarely the size of that package.
#
# THE RULE
#   An algorithm kata should never hit a wall this file could have avoided.
#   Anything a reasonable person writes while solving a problem belongs here.
#   Anything that talks to a network, a database, a filesystem beyond the
#   harness, or the Go toolchain itself does not.
